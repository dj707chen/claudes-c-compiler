Fix _Alignof parser to handle full abstract declarator suffixes (arrays, function pointers)

The _Alignof parser only handles pointer types (*) after the base type specifier,
but doesn't handle array dimensions ([N]) or other declarator suffixes that
sizeof already handles via parse_abstract_declarator_suffix.

This causes parse errors on valid C like:
  _Alignof(int[3])
  _Alignof(const int[5])
  _Alignof(struct_type[10])

The fix: replace the inline pointer-only loop in _Alignof with a call to
parse_abstract_declarator_suffix, matching sizeof's behavior.

Also fix the Alignof AST node and downstream consumers (sema, lowering)
to handle the richer type that parse_abstract_declarator_suffix can return.
