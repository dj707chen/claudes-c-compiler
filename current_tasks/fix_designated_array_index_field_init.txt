Fix designated array-index-plus-field initializers placing values at wrong offsets

When using C99 designated initializer syntax like:
  struct S table[5] = { [2].alloc = my_alloc, [3].free = my_free };

The compiler ignores the array index [N] and places values starting at index 0
instead of the correct index. For example, [2].alloc = my_alloc places the value
at table[0].alloc rather than table[2].alloc.

This affects global/static array initializers with combined array index + field
designators. The bug is in the global initialization lowering code.
