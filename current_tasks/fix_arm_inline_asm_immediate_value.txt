Fix ARM inline asm "i" constraint immediate values emitting 0

When classify_constraint returns AsmOperandKind::Immediate directly (for pure "i"/"n"
constraints), the imm_value field was never populated from Operand::Const, causing all
immediate values to be replaced with 0 in resolve_symbols_and_immediates.

This broke the ARM kernel build: `bic x9, x10, 0` instead of `bic x9, x10, 36028797018963968`
in __uaccess_mask_ptr (BIT(55) = 1UL << 55).

Status: in_progress
