Task: Fix i686 F64 (double) support
Status: in_progress (taken over from stalled worker)
Started: 2026-01-28

Description:
The i686 backend's F64 (double) handling is fundamentally broken because the
accumulator model uses eax (32-bit) but F64 needs 64 bits. This affects:

- F64 load/store through pointers (only copies 4 of 8 bytes)
- F64 binary operations (add, sub, mul, div) - only use low 32 bits
- F64 comparisons - use ucomiss (F32) instead of x87 fucomip
- F64 return values - use flds instead of fldl
- F64 negation - only flips low 32-bit sign bit
- F64 type conversions (F64<->F32, F64<->int)

Fix approach:
Treat F64 like I64/U64: use eax:edx pairs for 8-byte memory operations.
Use x87 FPU (fldl/fstpl) for F64 operations wherever needed.
Use fucomip for F64 comparisons.
Fix return path to use fldl for 8-byte double values.
