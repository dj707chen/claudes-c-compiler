Task: Fix conditional (ternary) expression type inference for arithmetic types

Problem:
sizeof(d ? c : 10l) returns 1 (sizeof char) instead of 8 (sizeof long).

The ExprTypeChecker::infer_expr_ctype for Expr::Conditional just returns the
type of the then-branch, ignoring the else-branch. Per C11 6.5.15, the result
type of a conditional expression with arithmetic operands should be the common
type after usual arithmetic conversions.

Affected files:
- src/frontend/sema/type_checker.rs (line ~264)
- src/ir/lowering/expr_types.rs (composite_conditional_type, line ~291)

Fix:
Apply CType::usual_arithmetic_conversion when both operands are arithmetic types.

Status: in_progress
