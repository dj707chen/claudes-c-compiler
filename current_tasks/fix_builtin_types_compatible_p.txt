Fix __builtin_types_compatible_p always returning 0

The parser currently discards both type arguments and returns IntLiteral(0).
This breaks signbit(), isgreater(), isless(), and other math.h macros that
use __builtin_choose_expr + __builtin_types_compatible_p for type-generic
dispatch.

Fix: Add a BuiltinTypesCompatibleP(TypeSpecifier, TypeSpecifier, Span) AST
node, and resolve it in the lowerer where we have type resolution for
__typeof__ expressions.
