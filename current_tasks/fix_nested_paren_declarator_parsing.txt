Fix nested parenthesized declarator parsing in function parameters

The parser fails on `int ((fnc)(int))` as a parameter type.
The nested-paren branch of parse_paren_param_declarator() calls
extract_paren_name() to extract '(fnc)', then immediately expects
')' for the outer parens, but '(int)' is in the way.

Fix: skip array dims and function param lists between the inner
name extraction and the outer ')' expectation.

Affects: compiler_suite_0139_0074 (and similar edge cases)
