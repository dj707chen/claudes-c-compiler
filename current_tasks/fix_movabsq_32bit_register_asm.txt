Task: Fix movabsq with 32-bit register names in inline asm
Status: in_progress
Started: 2026-01-28

Problem:
When a register-bound variable uses a 32-bit register name (e.g., `register uint32_t val asm("esi")`),
the inline asm emitter generates `movabsq $imm, %esi` which is invalid because movabsq requires
a 64-bit register operand. This causes assembly failures for tests like:
- asm-regvar-bitops-x86-005-run2
- asm-regvar-multi-x86-004-run2

Fix:
In load_input_to_reg (asm_emitter.rs), normalize 32-bit register names to 64-bit equivalents
when emitting 64-bit instructions (movabsq, xorq), or use 32-bit instructions (movl, xorl)
when the register is a 32-bit name.
