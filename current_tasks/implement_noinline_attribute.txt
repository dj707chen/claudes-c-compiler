Task: Implement __attribute__((noinline)) support
Status: in_progress
Started: 2026-01-26

Description:
Add support for the GCC __attribute__((noinline)) function attribute.
When present, the inlining pass must never inline the function, even if it
would otherwise be eligible (e.g., static inline).

This fixes test compiler_suite_0077_0033 which checks that noinline takes
precedence over always_inline when both are applied to the same function.

Changes:
- Parser: recognize "noinline" / "__noinline__" in GCC attributes
- AST: add is_noinline field to FunctionDef
- IR: add is_noinline field to IrFunction
- Lowering: propagate is_noinline from AST to IR
- Inline pass: skip functions with is_noinline in build_callee_map()
