Fix sizeof(string_literal) returning pointer size (8) instead of array
size in static/global initializer contexts.

Root cause: sema const_eval's sizeof_expr() calls infer_expr_ctype()
which returns CType::Pointer(Char) for string literals (applying
array-to-pointer decay). Per C11 6.3.2.1p3, this decay must not occur
inside sizeof. Fix: special-case StringLiteral and WideStringLiteral
in sema's sizeof_expr to return the correct array size.

This bug broke PostgreSQL's command tag length computation, causing
protocol message length mismatches that made all 216 regression tests fail.
