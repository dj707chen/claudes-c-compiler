Task: Fix struct array alloca sizing and char array string literal initialization

Bug 1: Struct array alloca sizing
Problem: When declaring local arrays of structs (e.g., struct Point pts[3]),
the alloca uses the struct's layout.size (8) instead of the full array
allocation size (24 = 3 * 8). This causes stack corruption and segfaults
when accessing elements beyond index 0.

Fix: In the actual_alloc_size computation, check is_array before using
struct layout size.

Bug 2: char[] = "string" initialization
Problem: Local char array initialization from string literals (e.g., char s[] = "ABC")
stores the string pointer instead of copying the string bytes into the array.

Fix: When initializing a char array from a StringLiteral, emit byte-by-byte
stores into the alloca instead of a single pointer store.

Status: Complete
