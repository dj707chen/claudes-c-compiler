Task: Implement _Thread_local / __thread TLS support
Status: in_progress

Problem:
_Thread_local and __thread are lexed and parsed but the annotation is silently
dropped. Thread-local variables are compiled as ordinary globals placed in
.data/.bss, causing incorrect behavior in multi-threaded programs.

Tests affected:
- compiler_suite_0057_0048 (SIGABRT - TLS not per-thread)
- compiler_suite_0057_0049 (SIGABRT - TLS not per-thread)

Plan:
1. Track is_thread_local from parser through AST Declaration and IR IrGlobal
2. Emit thread-local globals in .tdata/.tbss sections instead of .data/.bss
3. Generate TLS access patterns for x86-64 (Local Exec: %fs:offset@TPOFF)
4. Generate TLS access patterns for AArch64 (mrs tpidr_el0 + offset)
5. Generate TLS access patterns for RISC-V (tp register + offset)
6. Add regression test

TLS Model: Use Local Exec (simplest) which works for executables.
- x86-64: movl %fs:x@TPOFF, %eax
- AArch64: mrs x0, tpidr_el0; add x0, x0, #:tprel_hi12:x; ldr w0, [x0, #:tprel_lo12_nc:x]
- RISC-V: lui a0, %tprel_hi(x); add a0, a0, tp, %tprel_add(x); lw a0, %tprel_lo(x)(a0)
