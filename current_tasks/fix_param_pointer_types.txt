Fix parameter pointer/array type propagation

The parser's parse_param_declarator() consumes pointer stars (*) and array
brackets ([]) but discards this type information, only returning the parameter
name. This means `char *p` is stored as TypeSpecifier::Char instead of
TypeSpecifier::Pointer(Char), causing the lowering to treat pointer parameters
as their base type (1 byte for char* instead of 8 bytes for a pointer).

This causes segfaults in ~6500 tests where pointer parameters are truncated.

Fix:
1. Change parse_param_declarator to return (Option<String>, usize, bool)
   where the usize is pointer count and bool indicates array parameter
2. In parse_param_list, wrap type_spec in Pointer() layers based on pointer count
3. Array parameters decay to pointers
