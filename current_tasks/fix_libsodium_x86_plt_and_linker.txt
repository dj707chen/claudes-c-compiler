Fix libsodium x86 build: assembler jmp/jcc @PLT handling + linker @PLT symbol stripping

Root cause: When CCC compiles with -fPIC/-fPIE, it generates jmp/jcc foo@PLT instructions.
The assembler's encode_jmp and encode_jcc don't strip @PLT from label names or use PLT32 relocation.
This causes the linker to see symbols with @PLT suffix and fail to resolve them.

Fix:
1. Fix encode_jmp to strip @PLT and use R_X86_64_PLT32 (like encode_call already does)
2. Fix encode_jcc to strip @PLT and use R_X86_64_PLT32
3. Add linker-side stripping of @PLT from symbol names as defense-in-depth
4. Add regression tests
