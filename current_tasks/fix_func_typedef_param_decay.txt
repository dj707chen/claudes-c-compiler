Task: Fix bare function typedef parameter decay to pointer-to-function
Status: in_progress
Started: 2026-01-26

Bug: When a function parameter has a bare function typedef type (e.g.,
`typedef int filler_t(void*, void*); void foo(filler_t filler)`), the
compiler generates a direct call to an external symbol named after the
parameter instead of an indirect call through the function pointer.

Per C11 6.7.6.3p8, function-type parameters are adjusted to
pointer-to-function type, but this adjustment was missing for typedef-based
function types.

Root cause: Three missing pieces:
1. param_ctype() in types.rs only handles explicit fptr_params and
   func_ptr_typedefs, not bare function_typedefs
2. register_normal_param() in lowering.rs only registers ptr_sigs from
   fptr_params, not from function typedef info
3. As a result, is_func_ptr_variable() returns false and the call is
   emitted as a direct call

Fix: Added Case 3 to param_ctype() for bare function typedefs, and added
a fallback in register_normal_param() to register ptr_sigs from
function_typedefs when fptr_params is None.

Kernel file affected: mm/filemap.c (filemap_read_folio's filler_t parameter)
