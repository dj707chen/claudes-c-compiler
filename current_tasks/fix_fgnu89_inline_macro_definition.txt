Task: Fix -fgnu89-inline macro definition (__GNUC_GNU_INLINE__ vs __GNUC_STDC_INLINE__)
Status: in_progress
Started: 2026-01-30

Problem:
When -fgnu89-inline is passed, the compiler should define __GNUC_GNU_INLINE__ (and NOT
__GNUC_STDC_INLINE__). Currently, the compiler always defines __GNUC_STDC_INLINE__
regardless of the -fgnu89-inline flag.

This causes mpack and likely other projects to select the wrong inline linkage model,
leading to test failures because function pointer address comparisons fail when inline
functions get different definitions.

Fix: In the predefined macros, check for the -fgnu89-inline flag and define
__GNUC_GNU_INLINE__ instead of __GNUC_STDC_INLINE__ when it's set.

Affects: mpack (all 4 architectures) and potentially many other projects using
-fgnu89-inline with conditional compilation on __GNUC_GNU_INLINE__.
