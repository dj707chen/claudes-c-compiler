Task: Add support for %P and %a inline asm operand modifiers (x86)

Bug: The x86 inline asm template substitution does not recognize the %P or %a
operand modifiers. These are used extensively in the Linux kernel:

- %P[name]: Print raw operand value (symbol name without $ or % prefix).
  Used in "call %P[func]" to emit "call my_func" (not "call $my_func").

- %a[name]: Print operand as a memory address reference.
  Used in "%gs:%a[var]" for per-cpu variables, emitting the address offset.

These missing modifiers cause COMPILE_FAIL on virtually all kernel files because
the kernel headers (arch/x86/include/asm/alternative.h, percpu.h, etc.) use them.

Fix:
1. Add 'P' and 'a' to the modifier character list in substitute_x86_asm_operands()
2. Handle %P: emit operand value without $ or % prefix (like %c but for uppercase)
3. Handle %a: emit operand as memory reference (address form)
4. Add regression tests
