Task: Implement __attribute__((alias("target"))) and __attribute__((weak))

musl libc relies heavily on weak_alias(old, new) which expands to:
  extern __typeof(old) new __attribute__((__weak__, __alias__(#old)));

This creates a weak symbol alias. Our compiler currently silently skips
both alias and weak attributes in parse_gcc_attributes().

Implementation plan:
1. Parser: extract alias target string and weak flag from attributes
2. AST: add alias_target (Option<String>) and is_weak (bool) to InitDeclarator
3. IR: add aliases Vec to IrModule: (alias_name, target_name, is_weak, is_function)
4. Lowering: when a declaration has alias_target, add to module aliases
5. Backend: emit .weak/.globl and .set directives for aliases

This is needed by 199+ files in musl, and is the primary reason musl
test programs fail to link (___errno_location and hundreds of other
aliases are undefined).

Status: In progress
